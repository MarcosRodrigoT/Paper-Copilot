# Step 8: Streamlit UI

This document explains the Streamlit web interface and how it connects to the agent.

## What is Streamlit?

**Streamlit** is a Python framework for building web apps with minimal code. You write a regular Python script, call Streamlit functions to create UI elements, and Streamlit turns it into an interactive web app.

Key idea: **Streamlit re-runs your entire script** every time the user interacts with the UI (clicks a button, uploads a file, etc.). This is different from frameworks like Flask where you define route handlers.

```python
import streamlit as st

st.title("My App")
name = st.text_input("Your name")
if st.button("Greet"):
    st.write(f"Hello, {name}!")
```

Run with: `streamlit run app.py`

## How the UI connects to the agent

The connection is simple — `app.py` imports and calls `process_paper()` from `src/agent.py`:

```python
from src.agent import process_paper

# When user clicks "Generate Summary":
result_path = process_paper(str(input_path), model_name=model_name)
```

### The flow

1. **User uploads PDF** → Streamlit saves it to `input/` directory
2. **User clicks "Generate Summary"** → `process_paper()` is called
3. **Agent runs** → creates output in `output/<paper_name>/`
4. **UI reads** `output/<paper_name>/summary.md` and displays it
5. **Download buttons** let user get the markdown or a zip with images

### File handling

When a file is uploaded through Streamlit, it exists only in memory. We save it to the `input/` folder so:
- The agent can access it by file path (tools need a path, not bytes)
- It persists if you want to re-process it later
- The CLI mode (`python -m src.agent`) can use the same files

## Key Streamlit features used

### `st.file_uploader`
Creates the drag-and-drop area. The `type=["pdf"]` parameter restricts uploads to PDFs.

### `st.status`
Shows a progress indicator while the agent is working. It has three states:
- Running (spinner)
- Complete (checkmark)
- Error (X)

### `st.download_button`
Creates a button that triggers a file download. We use two:
- **Download Markdown** — just the `.md` file
- **Download All (ZIP)** — the `.md` + all images in a zip archive

### `st.sidebar`
The settings panel on the left. Contains the model selector and usage instructions.

### `st.markdown`
Renders the generated summary as formatted markdown directly in the browser.

## Running the UI

```bash
# Start the Streamlit server
uv run streamlit run app.py

# It will open in your browser at http://localhost:8501
```

## Customization

- **Change the model:** Edit the text input in the sidebar (or change `OLLAMA_MODEL` in `src/config.py`)
- **Change the port:** `uv run streamlit run app.py --server.port 8080`
- **Headless mode:** `uv run streamlit run app.py --server.headless true` (no browser auto-open)
